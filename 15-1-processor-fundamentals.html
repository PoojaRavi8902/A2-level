Central Processing Unit (CPU)
The CPU is often called the brain of the computer. It performs all the calculations and decisions, and controls how information flows between memory and devices. Think of it as a manager who directs all operations inside the computer.

  • ALU (Arithmetic Logic Unit): Performs arithmetic (add, subtract) and logic (AND, OR, NOT) operations.
  • CU (Control Unit): Tells the computer what to do next ‚Äî fetch, decode, execute instructions.
  • Registers: Tiny storage inside the CPU for very fast data access.
  • System Bus: Connects CPU, memory, and input/output devices.

Types of Processor Architecture

There are two main processor types ‚Äî CISC and RISC. Let‚Äôs understand how they differ.

CISC (Complex Instruction Set Computer)

Has many complex instructions. One command can do several things at once. For example, ‚ÄúADD A, B, C‚Äù can load data, add them, and store the result ‚Äî all in one instruction.

  • Advantages: Easier for software, shorter programs.
  • Disadvantages: Hardware is complex, slower execution per instruction.

RISC (Reduced Instruction Set Computer)

Uses fewer, simpler instructions. Each instruction does only one small task but executes quickly. Example: ARM processors found in phones and tablets.

  • Advantages: Fast, efficient, simpler design, good for mobile devices.
  • Disadvantages: Needs more lines of code to complete a task.

üîπ 2. Interrupt Handling

An interrupt is a signal that tells the CPU to stop what it‚Äôs doing and handle something urgent. For example, when you press a keyboard key, the CPU pauses its current work to read that key press.

Steps:

  1. CPU finishes current instruction.
  2. It saves its current state (registers, program counter).
  3. It jumps to a special routine ‚Äî the Interrupt Service Routine (ISR).
  4. After handling the event, it goes back to what it was doing.

Types:

  • Hardware interrupt: Comes from devices like keyboard, printer.
  • Software interrupt: Created by programs or errors (e.g., division by zero).

üîπ 5. Virtual Machines

A Virtual Machine (VM) acts like a computer inside your computer. It lets you run another operating system (like Linux on a Windows PC) without affecting the main one.

How it works: Hypervisor software manages different operating systems on the same physical hardware.

Types:

  • System VM: Runs a complete OS (e.g., Windows + Ubuntu).
  • Process VM: Runs one program across platforms (e.g., Java Virtual Machine).

Advantages: Easy testing, isolation, and hardware efficiency.
Disadvantages: Slower performance, more memory use.

üßæ Summary
  • RISC and CISC define how CPUs execute instructions.
  • Interrupts allow urgent events to be handled immediately.
  • Pipelining boosts performance by overlapping instruction stages.
  • Parallel processing improves speed using multiple cores.
  • Virtual Machines simulate other computers for testing or flexibility.
