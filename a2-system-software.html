üß† Memory Management, Paging, and Virtual Memory

The OS allocates, tracks, and protects memory use for efficient multiprogramming.

  • Segmentation: Divides program into variable-size parts.
  • Paging: Divides both program and memory into equal-sized blocks.
  • Virtual Memory: Uses disk space to extend available memory.
üìò Disk Thrashing occurs when paging activity dominates CPU time, degrading performance.

  • Interpreter: Line-by-line execution
‚úÖ Exam Tip: Compilers are faster but harder to debug; interpreters are slower but easier to test.

üíª System Utilities

Utility software supports maintenance and optimization of system performance.

  • Anti-Virus: Detects and removes malware
  • Disk Management: Partitioning, defragmentation, error-checking

üß∞ Device Drivers and OS-Hardware Communication

Device Drivers are specialized software that act as intermediaries between the operating system and hardware devices.

üîπ What Are Device Drivers?

They are software modules that translate general OS instructions into specific commands that hardware can understand.

‚öôÔ∏è Role in OS-Hardware Communication

  • Enable the OS to use peripherals (printer, mouse, display).
  • Handle hardware-level interrupts and responses.
  • Abstract hardware complexity ‚Äî OS doesn‚Äôt need to know hardware details.
üí° Example:
A printer driver converts OS print commands into device-specific language like PCL or PostScript.

üìò Key Points

  • Each hardware device needs a specific driver.
  • Drivers are loaded during boot or when the device is connected.
  • Modern OSs use plug-and-play for automatic driver installation.
üß† Exam Tip: Drivers ensure modularity ‚Äî OS doesn‚Äôt need rewriting for every new hardware model.
