Chapter 19 — Computational Thinking & Problem Solving

Teacher-friendly 60-minute lesson • Focus: practical exam skills, pseudocode, test cases, trace tables
Lesson length
60 minutes (15 | 10 | 15 | 12 | 8)

Phase 1 — Starter (15 minutes)

Goal: Activate prior knowledge (algorithms, data representation, simple programming constructs) and surface misconceptions.

  1. Quick written warm-up (8 min): 6 short tasks on the board — binary to decimal, simple pseudocode trace, identify loop type, name a data structure, give one test case, spot the error in a 3-line pseudocode snippet.
  2. Pair discussion (4 min): swap answers, explain one correction to partner.
  3. Teacher check (3 min): sample 3 answers aloud and note class-wide issues.
Teacher notes (script):
  1. "We’ll do a short warm-up to prime your algorithmic thinking — don’t worry about perfect answers."
  2. If many students miss the same item, plan to add a 2‑minute clarification during Lecture 1.
  3. Collect one good answer from each pair for quick formative assessment."
Starter tasks (write answers on paper or in the chat)
  1. Convert binary 10110 to decimal.
  2. What does this pseudocode output? 
    FOR i ← 1 TO 3\n  PRINT i\nENDFOR
  3. Identify the data structure: push, pop operations.
  4. Give a normal test and a boundary test for reading a student's mark (0–100).
  5. Spot the bug: 
    IF x = 10 THEN\n  PRINT "Ten"\nENDIF\nELSE\n  PRINT "Not Ten"\n
  6. Which algorithm is best for sorted data searching? (one-word answer)

Phase 2 — Lecture 1 (10 minutes)

Key Concepts & Strategies (19.2)

  1. Decomposition — break problems into independent functions (show student grading system decomposition).
  2. Abstraction — ignore irrelevant detail; use clear interfaces.
  3. Pattern recognition — reuse known solutions.
  4. Algorithm design — step-by-step with clear pseudocode conventions.
  5. Testing & debugging — normal, boundary, invalid tests; trace tables and dry runs.
What to say (script):
  1. "Start by decomposing — write small functions, then integrate them."
  2. "Always design test cases before coding — it saves time in debugging."
  3. "Abstraction helps you ignore file formats or UI when designing core logic."

Pseudocode conventions (19.3) — display and explain:

DEFINE calculateAverage(marks[1..n])\n  total ← 0\n  FOR i ← 1 TO n\n    total ← total + marks[i]\n  ENDFOR\n  RETURN total / n\nENDDEFINE
Teacher note: remind students to state indexing (0‑ or 1‑based) and variable names clearly.

Phase 3 — Activity (15 minutes)

Group practical tasks — trace tables & debugging

  1. Split into groups of 3. Each group gets one of the following in printed form (or projected):
    • Task A: Work the trace table for findMax on [3,7,2,9,4] and show intermediate steps.
    • Task B: Complete the trace table for one pass of the bubble-like pseudocode (given in 19.11 Q7).
    • Task C: Debug this pseudocode (find the logic error and correct it):
      IF x % 2 = 0 THEN\n  PRINT "Odd"\nELSE\n  PRINT "Even"\nENDIF
  2. Groups swap papers and peer-mark using the provided mark scheme (quick feedback).
  3. Teacher circulates and gives 1‑minute targeted feedback to two groups showing common errors.
Marking tips (quick):
  • Trace table completeness (2 marks), correct updates (2 marks), final answer (1 mark).
  • For debugging: identify the incorrect condition and provide corrected code (3 marks).
Activity Pack — printable snippets
  1. findMax trace: table header: i | arr[i] | maxVal | maxIndex | comments
  2. Bubble pass: initial [4,2,7,3] -> please show swaps for i=1..3
  3. Debug: the if condition prints wrong labels; correct to IF x % 2 = 0 THEN PRINT "Even" ELSE PRINT "Odd" ENDIF

Phase 4 — Lecture 2 (12 minutes)

Worked examples: Searching, Sorting & Full Practical Problem

Linear vs Binary Search (19.5)

DEFINE binarySearch(arr[1..n], key)\n  low ← 1\n  high ← n\n  WHILE low ≤ high\n    mid ← (low + high) DIV 2\n    IF arr[mid] = key THEN\n      RETURN mid\n    ELSEIF arr[mid] < key THEN\n      low ← mid + 1\n    ELSE\n      high ← mid - 1\n    ENDIF\n  ENDWHILE\n  RETURN -1\nENDDEFINE
Explain: binary search O(log n), use only on sorted arrays.

Selection Sort (19.6)

DEFINE selectionSort(arr[1..n])\n  FOR i ← 1 TO n-1\n    minIndex ← i\n    FOR j ← i+1 TO n\n      IF arr[j] < arr[minIndex] THEN minIndex ← j ENDIF\n    ENDFOR\n    swap arr[i] and arr[minIndex]\n  ENDFOR\n  RETURN arr\nENDDEFINE

Full Practical: Student Grade Report (19.9)

Walk through decomposition, pseudocode for assignGrade and calculateStats, and show trace on sample input.

DEFINE assignGrade(mark)\n  IF mark >= 70 THEN RETURN 'A'\n  ELSEIF mark >= 60 THEN RETURN 'B'\n  ELSEIF mark >= 50 THEN RETURN 'C'\n  ELSEIF mark >= 40 THEN RETURN 'D'\n  ELSE RETURN 'F'\n  ENDIF\nENDDEFINE
Teaching tip: show a worked trace table for the sample 4 students (avg=71, highest=92, lowest=49).

Chapter 19 — Full Notes (condensed)

Computational thinking is the process of formulating problems and solutions so a computer can execute them. Key skills include decomposition, abstraction, pattern recognition, algorithm design, testing and complexity analysis.

  2. Short answers (count even numbers, boundary tests, trace table uses)
