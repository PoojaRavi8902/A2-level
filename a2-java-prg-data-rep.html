1. Variables and Constants

In Java, every variable must have a declared type. Constants are declared using final. Data types are broadly categorized into:

  • Primitive types: int, double, char, boolean, etc.
  • Reference types: Strings, Arrays, Objects.

    System.out.println("Pi: " + PI);
    System.out.println("Grade: " + grade);

14.04 Boolean Expressions

Boolean expressions are conditions that evaluate to either true or false. Java supports:

Comparison Operators

  • == ‚Üí equal to
  • != ‚Üí not equal to
  • > ‚Üí greater than
  • < ‚Üí less than
  • >= ‚Üí greater than or equal to
  • <= ‚Üí less than or equal to

Boolean Operators

  • && ‚Üí AND
  • || ‚Üí OR
  • ! ‚Üí NOT

// Example of Boolean expressions in Java
public class Main {
    public static void main(String[] args) {
        int x = 10, y = 20;

        System.out.println(x == y);   // false
        System.out.println(x < y);    // true
        System.out.println((x < y) && (y > 15)); // true
        System.out.println(!(x > y)); // true
    }
}

Example: Days in a Month (with leap year check)


import java.util.Scanner;

public class DaysInMonth {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        System.out.print("Enter month number (1-12): ");
        int month = input.nextInt();
        System.out.print("Enter year: ");
        int year = input.nextInt();

        int days = 0;
        switch (month) {
            case 1: case 3: case 5: case 7: case 8: case 10: case 12:
                days = 31; break;
            case 4: case 6: case 9: case 11:
                days = 30; break;
            case 2:
                days = 28;
                if (year % 400 == 0 || (year % 4 == 0 && year % 100 != 0)) {
                    days = 29;
                }
                break;
            default:
                System.out.println("Invalid month number");
        }
        if (days > 0) {
            System.out.println("Days: " + days);
        }
    }
}

14.13 Putting it all together

In this section, we bring together the concepts of functions, procedures, and parameters by writing a complete Java program that draws a pyramid of symbols.

Notice how the parameters of the methods have identifiers that differ from the variables in main(). This helps make clear that parameters and local variables inside methods are separate from those in the calling program.

Java Pyramid Drawing Program


import java.util.Scanner;

public class Pyramid {
    static final char SPACE = ' ';

    public static char getSymbol() {
        Scanner console = new Scanner(System.in);
        System.out.print("What symbol do you want to use? ");
        String response = console.next();
        return response.charAt(0);
    }

    public static int inputMaxNumberOfSymbols() {
        Scanner console = new Scanner(System.in);
        int number = 0;
        while ((number % 2) == 0) {
            System.out.print("How many symbols make the base? (odd number): ");
            number = console.nextInt();
        }
        return number;
    }

    public static void outputChars(int number, char symbol) {
        for (int count = 0; count < number; count++) {
            System.out.print(symbol);
        }
    }

    public static int adjustSpacesForNextRow(int spaces) {
        return spaces - 1;
    }

    public static int adjustSymbolsForNextRow(int symbols) {
        return symbols + 2;
    }

    public static void main(String[] args) {
        char thisSymbol = getSymbol();
        int maxNumberOfSymbols = inputMaxNumberOfSymbols();
        int numberOfSpaces = (maxNumberOfSymbols + 1) / 2;
        int numberOfSymbols = 1;

        while (numberOfSymbols <= maxNumberOfSymbols) {
            outputChars(numberOfSpaces, SPACE);
            outputChars(numberOfSymbols, thisSymbol);
            System.out.println();
            numberOfSpaces = adjustSpacesForNextRow(numberOfSpaces);
            numberOfSymbols = adjustSymbolsForNextRow(numberOfSymbols);
        }
    }
}

Discussion Point: Notice how the two procedures OutputSpaces and OutputSymbols have been replaced by a single method outputChars() without changing the program‚Äôs effect.

14.15 Text Files

In Java, text file handling is done using classes from the java.io package. You can write data to files, read data back, append to existing files, and process until the end-of-file (EOF) is reached.

Writing to a Text File

To write to a text file, you typically use FileWriter combined with PrintWriter. If the file already exists, writing will overwrite it unless you explicitly open it in append mode. 


import java.io.FileWriter;
import java.io.PrintWriter;
import java.io.IOException;

public class WriteExample {
    public static void main(String[] args) throws IOException {
        FileWriter fileHandle = new FileWriter("SampleFile.txt", false); // false = overwrite
        PrintWriter printLine = new PrintWriter(fileHandle);

        String lineOfText = "Hello, this is my first line!";
        printLine.printf("%s%n", lineOfText);

        printLine.close();
        System.out.println("File written successfully.");
    }
}
Note: Always close the file after writing, otherwise data may not be saved properly.

Reading from a Text File

To read from a text file, use FileReader wrapped in BufferedReader. Each call to readLine() returns one line, or null when the end of the file is reached. 


import java.io.FileReader;
import java.io.BufferedReader;
import java.io.IOException;

public class ReadExample {
    public static void main(String[] args) throws IOException {
        FileReader fileHandle = new FileReader("SampleFile.txt");
        BufferedReader textReader = new BufferedReader(fileHandle);

        String lineOfText = textReader.readLine();
        while (lineOfText != null) {
            System.out.println(lineOfText);
            lineOfText = textReader.readLine();
        }

        textReader.close();
    }
}
Tip: You can also use Scanner for simple file reading, but BufferedReader is faster for larger files.

Appending to a Text File

To append data without overwriting existing content, set the second parameter of FileWriter to true. 


import java.io.FileWriter;
import java.io.PrintWriter;
import java.io.IOException;

public class AppendExample {
    public static void main(String[] args) throws IOException {
        FileWriter fileHandle = new FileWriter("SampleFile.txt", true); // true = append
        PrintWriter printLine = new PrintWriter(fileHandle);

        String lineOfText = "This line will be added to the end.";
        printLine.printf("%s%n", lineOfText);

        printLine.close();
        System.out.println("Line appended successfully.");
    }
}

End-of-File (EOF)

In Java, there is no explicit EOF marker. Instead, methods such as readLine() return null when the end of the file is reached. 


import java.io.FileReader;
import java.io.BufferedReader;
import java.io.IOException;

public class EOFExample {
    public static void main(String[] args) throws IOException {
        BufferedReader reader = new BufferedReader(new FileReader("Test.txt"));
        String line = reader.readLine();

        while (line != null) {
            System.out.println(line);
            line = reader.readLine();
        }

        reader.close();
    }
}
