17.01 Transmission Modes

In computer networking, data transmission is the process of sending and receiving data between two or more digital devices. Two broad approaches are commonly used: circuit switching and packet switching. Each approach suits different kinds of communication.

Circuit Switching

Circuit switching establishes a dedicated communication path between sender and receiver before any data is sent. The path is reserved for the duration of the session which guarantees constant bandwidth and predictable latency.

Steps in circuit-switched transmission:

  1. The sender identifies the receiver’s address or identity.
  2. The system checks whether the receiver is available to receive data.
  3. A dedicated communication path is established through network switches and exchanges.
  4. Data is transmitted through this path.
  5. Once communication ends, the path is released for other users.

Circuit switching is ideal for continuous real-time traffic (historically voice). It is inefficient for intermittent data because the reserved channel stays idle during pauses.

Worked example:

Consider a video conference between two offices using a leased line. When the call begins a dedicated circuit is established, guaranteeing high-quality, uninterrupted audio and video. If participants stay silent for minutes, the line remains reserved — reliable performance at the cost of wasted bandwidth during idle time.

Packet Switching

Packet switching divides data into packets. Each packet is sent independently, may take different routes, and is reassembled at the destination. No dedicated path is reserved — this enables efficient sharing of network resources and is the basis of the Internet.

There are two modes in packet-switched communication:

  • Connectionless (e.g., UDP): packets are sent without establishing a prior connection and without guaranteed delivery.
  • Connection-oriented (e.g., TCP): a logical connection is used, packets are acknowledged and lost packets retransmitted to ensure reliability.
Worked example:

Sending an email: the message is split into packets. Each packet may take different routes depending on current network traffic. Routers inspect headers to find the best next hop. At the destination the packets are reassembled to reconstruct the full message.

17.02 Protocols

A protocol is a standardized set of rules governing data transmission between devices on a network. Protocols specify message formats, addressing, timing, error checking and recovery methods. They allow devices with different hardware and software to communicate reliably.

Key term: Protocol — a set of rules for data transmission agreed upon by both sender and receiver.

Worked example:

When a web browser requests a webpage it uses the HTTP protocol. The browser formats a request message, the server interprets it and returns a response following HTTP rules so the requested page is correctly delivered.

Why protocols matter
  • Interoperability: allow devices from different vendors to work together.
  • Reliability: define how errors are detected and corrected.
  • Scalability: define addressing and routing so networks can grow.

17.03 A Protocol Stack

A protocol stack organizes network functions into layers. Each layer handles a particular responsibility and provides services to the layer above while depending on the layer below. This modular design simplifies implementation and troubleshooting.

Key concept: each layer provides services to the layer above and relies on services from the layer below.

17.04 The TCP/IP Protocol Suite

The TCP/IP suite is the basis for Internet communications. Typical layerings show Application, Transport, Internet (Network) and Link (Data Link + Physical) layers. Important protocols include TCP and IP.

Transmission Control Protocol (TCP)

TCP guarantees reliable, ordered delivery of data. It:

  • Splits data into segments and assigns sequence numbers.
  • Uses acknowledgements to confirm receipt.
  • Retransmits lost segments and performs flow control (sliding windows).
Worked example:

When downloading a file, TCP numbers packets. If packet 5 is lost, the receiver asks for retransmission so the file is complete and in order.

Internet Protocol (IP)

IP is responsible for addressing and routing packets through networks. Each device is assigned an IP address. IP is connectionless — packets are forwarded based on header information and routing tables without guarantees of delivery.

Exam tip: TCP focuses on reliability (ordering, retransmission); IP focuses on routing (finding paths between networks). Remember: together they are the core of Internet communication.

17.05 The Ethernet Protocol Stack

Ethernet is a family of protocols used in Local Area Networks (LANs). It implements the lower layers (Data Link and Physical) in the TCP/IP model and specifies how frames are formatted and how devices share the medium.

Ethernet sublayers include Logical Link Control (LLC), Media Access Control (MAC), Physical Coding Sublayer (PCS), and Physical Medium Attachment (PMA). The MAC layer uses addresses (MAC addresses) to deliver frames to the correct hardware device.

Worked example:

In a school LAN, when a teacher sends a file to a network printer, Ethernet packages the data into frames addressed to the printer's MAC address and transmits them over the cable until the printer receives and processes the frames.

17.06 Application-Layer Protocols

Application-layer protocols define how user applications talk across the network. Common examples include HTTP, FTP, SMTP, POP3 and IMAP.

HTTP (HyperText Transfer Protocol)

HTTP is the protocol of the World Wide Web. It follows a client–server model: a browser sends requests (GET, POST, PUT, etc.) and the web server responds with content or an error. HTTP messages are plain-text ASCII formatted requests and responses.

FTP (File Transfer Protocol)

FTP is designed to transfer files between computers. It uses separate control and data channels and supports commands for directory navigation, upload and download. Commonly used to upload website files to hosting servers.

Email protocols — SMTP, POP3, IMAP

SMTP (Simple Mail Transfer Protocol) sends email between mail servers and from clients to servers for outgoing mail. POP3 downloads messages from the mail server to a single device (often removing them from the server). IMAP keeps messages on the server and allows access from multiple devices, syncing state (read/unread folders) between clients.

17.07 Peer-to-Peer (P2P) File Sharing

Peer-to-peer networks let users share files directly without a central server. Each peer acts as both client and server. BitTorrent is a widely used protocol that breaks files into small pieces (chunks) and downloads/uploads simultaneously with many peers — improving speed and resilience.

Worked example:

Downloading a large file via BitTorrent: your client connects to many peers and obtains different chunks from each. As you receive chunks you also upload them to others. This distributed approach speeds up downloads and reduces load on any single source.
