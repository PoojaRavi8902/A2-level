TEACHER PANE

Section 19 – Computational Thinking & Problem-Solving

Purpose of this Chapter

Section 19 prepares students for the thinking behind programming. This is not about syntax — it is about how Cambridge expects students to design, reason, and justify solutions.

Exam reality:
Students who fail Paper 4 usually do not fail because of syntax. They fail because they cannot:
  • Break problems down
  • Choose correct logic
  • Justify recursion

19.1 Algorithms

Teacher Conceptual Focus

Cambridge defines an algorithm as a finite, ordered sequence of unambiguous steps. Students must understand that:

  • An algorithm exists before code
  • It must be language-independent
  • It must terminate
Cambridge wording you should drill:
“Design an algorithm to…”
“Express the algorithm using structured English or pseudocode”

Designing Algorithms to Solve Problems

Teach students a repeatable mental process:

  1. Understand the problem fully
  2. Identify inputs and outputs
  3. Break into steps
  4. Decide selection and repetition
  5. Check edge cases
Common examiner comment:
“Candidate jumped straight to code without designing an algorithm.”

Expressing Algorithms in Structured Form

Cambridge accepts:

  • Structured English
  • Pseudocode
  • Flowcharts

But **structured pseudocode is preferred** in Paper 4.

Using Logic Statements

You must explicitly teach:

  • IF
  • IF…ELSE
  • Nested IF
  • FOR loops
  • WHILE loops
Teaching tip:
Always ask: “How many times do I know this will repeat?”

Section 19 – Algorithms

What is an Algorithm?

An algorithm is a clear set of steps used to solve a problem. It is written so that any programmer can follow it.

EXAM DEFINITION (MEMORISE):
An algorithm is a finite sequence of well-defined steps used to solve a problem.

Designing an Algorithm

Before writing any code, always answer:

  • What data do I receive?
  • What result do I produce?
  • What decisions must I make?

Example: Find the Largest Number

Problem: Input three numbers and output the largest. 
SET max ← first number
IF second number > max
    SET max ← second number
IF third number > max
    SET max ← third number
OUTPUT max
Why this scores marks:
Clear logic, structured steps, no programming language.

Logic Statements

IF / ELSE

Used when a decision must be made. 

IF mark >= 50
    OUTPUT "Pass"
ELSE
    OUTPUT "Fail"

FOR Loop

Used when the number of repetitions is known. 

FOR i ← 1 TO 10
    OUTPUT i
NEXT i

WHILE Loop

Used when repetition depends on a condition. 

WHILE password ≠ "correct"
    INPUT password
ENDWHILE
Exam trap:
Using FOR when the number of repeats is unknown.

Practice Questions

1. Design an algorithm to calculate the average of 10 numbers.
2. Design an algorithm to count how many values are greater than 50.
3. Explain why a WHILE loop is suitable for password entry.
Self-check:
Did you:
  • Use clear steps?
  • Avoid programming syntax?
  • Show logic clearly?

19.2 Recursion

Why Recursion Is Difficult for Students

Recursion is one of the most poorly understood topics at A Level. Most students can write a recursive function but cannot explain how it works — which is exactly what Cambridge tests.

Examiner observation:
“Candidates often write a recursive solution without identifying a base case, leading to infinite recursion.”

Definition and Purpose of Recursion

Recursion is a technique where a problem is solved by breaking it down into smaller versions of the same problem. A recursive algorithm is one that calls itself.

Cambridge-approved definition:
Recursion is a programming technique in which a procedure or function calls itself.

Purpose of Teaching Recursion

  • To solve problems with repetitive sub-structure
  • To replace complex iteration in some cases
  • To model problems that naturally repeat (trees, fractals, divide-and-conquer)
Teacher emphasis:
Recursion is not “better than loops” — it is appropriate in specific situations.

Essential Features of Recursion

Every correct recursive solution MUST contain:

  1. A base case
  2. A recursive case
  3. Progress toward the base case
Absolute exam rule:
No base case = zero credit for recursion explanation.

Base Case

The base case is the condition that stops the recursion. It prevents infinite calls.

Recursive Case

The recursive case is where the function calls itself with a simpler input.

Cambridge phrasing to model:
“The base case stops further recursive calls.”

How Recursion Operates (Call Stack)

This is the most examinable part of recursion. Students must understand that:

  • Each recursive call is stored on the call stack
  • Local variables are stored separately for each call
  • Functions return values in reverse order
Teaching method:
Physically draw the stack on the board. Do not skip this.

Stack Behaviour

  • Last call made is the first to complete
  • Matches LIFO principle
  • Very similar to stack data structure
Examiner trap:
Students describe recursion but do not mention the call stack.

When to Use Recursion

Recursion is suitable when:

  • The problem can be divided into smaller identical problems
  • The stopping condition is clear
  • The solution is naturally hierarchical

Typical Exam Examples

  • Factorial
  • Fibonacci sequence
  • Searching tree structures
  • Traversing nested data

When NOT to Use Recursion

  • When iteration is simpler
  • When performance is critical
  • When stack memory may overflow
High-mark explanation:
Recursion can be inefficient due to repeated function calls and stack usage.

STUDENT PANE

Recursion (19.2)

What Is Recursion?

Recursion is when a function solves a problem by calling itself. Each call works on a smaller version of the problem.

EXAM DEFINITION:
Recursion is a programming technique where a function calls itself.

The Two Required Parts

1. Base Case

The base case is the condition that stops the recursion.

2. Recursive Case

The recursive case is where the function calls itself with a simpler value.

Exam warning:
If you forget the base case, the program will never stop.

Example: Factorial

Factorial of a number n is: 

n! = n × (n - 1)!

Pseudocode

FUNCTION factorial(n)
    IF n = 0
        RETURN 1
    ELSE
        RETURN n * factorial(n - 1)
END FUNCTION
Why this scores full marks:
  • Clear base case
  • Recursive call simplifies the problem
  • Correct return value

How Recursion Uses Memory

Each recursive call is placed on the call stack.

Example: factorial(3)

factorial(3)
→ factorial(2)
→ factorial(1)
→ factorial(0)

When the base case is reached, values return back up the stack.

Order of returns:
factorial(0) → factorial(1) → factorial(2) → factorial(3)

Tracing Recursion (Exam Skill)

Cambridge often asks you to:

  • Trace recursive calls
  • Show return values
  • Explain stack behaviour

Practice Trace

FUNCTION sum(n)
    IF n = 0
        RETURN 0
    ELSE
        RETURN n + sum(n - 1)
END FUNCTION

What is the output of sum(3)?

Answer:
3 + 2 + 1 + 0 = 6

When Should You Use Recursion?

  • When the problem repeats itself
  • When the solution is easier to understand recursively
  • When each step reduces the problem size

When Should You Avoid Recursion?

  • When loops are simpler
  • When performance matters
  • When memory is limited
Exam trap:
Never say “recursion is always better than iteration”.

Exam-Style Practice

1. Define recursion. 2. Explain the purpose of a base case. 3. Trace the output of a recursive function with input 4. 4. State one advantage and one disadvantage of recursion.
Self-check before exam:
  • Can I identify the base case?
  • Can I explain stack behaviour?
  • Can I trace recursive calls?

Exam Mastery – Section 19 (Paper 4 Focus)

How Section 19 Is Examined

Section 19 knowledge is examined primarily in Paper 4 (Practical), but elements also appear in Paper 3 explanations.

  • Algorithm design before coding
  • Correct use of selection and iteration
  • Appropriate use of recursion
  • Clear logic and structure
Examiner priority:
Marks are awarded for thinking, not typing speed.

Typical Paper 4 Question Pattern

  1. Problem description
  2. Inputs and outputs defined
  3. Requirement to design or complete an algorithm
  4. Implementation in a chosen language
  5. Testing or explanation
Common failure:
Students jump straight into code without a plan.

Teacher Diagnostic Checklist

  • Can the student design before coding?
  • Can they choose IF vs WHILE correctly?
  • Can they justify recursion?
  • Can they trace recursive calls?
Intervention tip:
If a student cannot trace recursion on paper, do not allow them to code it.

Mark Allocation Reality

Cambridge awards marks for:

  • Correct logic
  • Correct structure
  • Appropriate control flow
  • Clear termination conditions
Zero marks scenario:
Recursive function without a base case.

Teacher Pacing Guide

  • Algorithms: 4–6 lessons
  • Logic & loops: 3–4 lessons
  • Recursion: 5–6 lessons
  • Exam mastery & practice: 4 lessons

Exam Mastery – Section 19

Paper 4 Practical Task – Example 1

Task:
Design an algorithm that inputs 10 numbers and outputs how many are greater than 50.

Step 1: Identify Inputs & Outputs

  • Input: 10 numbers
  • Output: count of numbers > 50

Step 2: Algorithm (Pseudocode)

SET count ← 0
FOR i ← 1 TO 10
    INPUT number
    IF number > 50
        count ← count + 1
    ENDIF
NEXT i
OUTPUT count
Why this scores full marks:
  • Correct loop choice
  • Clear condition
  • Correct counting logic

Paper 4 Practical Task – Example 2 (Recursion)

Task:
Write a recursive algorithm to calculate the sum of numbers from 1 to n.

Algorithm

FUNCTION sum(n)
    IF n = 0
        RETURN 0
    ELSE
        RETURN n + sum(n - 1)
END FUNCTION
Examiner comments:
  • Base case clearly stated
  • Recursive case simplifies problem
  • Terminates correctly

Tracing Question (Very Common)

FUNCTION mystery(n)
    IF n = 1
        RETURN 1
    ELSE
        RETURN n * mystery(n - 1)
END FUNCTION

What is the output of mystery(4)?

Answer:
4 × 3 × 2 × 1 = 24
Exam trap:
Forgetting that recursive calls return values.

Mini Test (Exam Conditions)

Answer without notes:
  1. Define recursion.
  2. State the purpose of a base case.
  3. Design an algorithm to find the smallest of 5 numbers.
  4. Explain one advantage of recursion.
  5. Explain one disadvantage of recursion.
Model Points:
  • Function calling itself
  • Stops infinite recursion
  • Clear selection logic
  • Elegant solution for repetitive problems
  • Uses extra memory / stack

Common Examiner Mistakes

  • No base case in recursion
  • Wrong loop choice
  • Unclear algorithm steps
  • Using programming syntax in algorithm questions
Critical warning:
Do NOT write Python/Java when asked for an algorithm.

Final Exam Survival Checklist

  • I can design algorithms before coding
  • I can choose IF, FOR, or WHILE correctly
  • I can explain recursion clearly
  • I can trace recursive calls
  • I always include a base case
If you can tick all of these, you are exam-ready.
