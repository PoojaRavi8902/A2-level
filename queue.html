Queues in Python (Using LIST) – Cambridge AS Level 9618

Teacher Pane – Explanation & Teaching Flow

1. What is a Queue?

A queue is a linear data structure that follows the rule:

FIFO – First In, First Out

This means the first item added is the first item removed.

Textbook analogy: students standing in a line.
2. Why use Python LIST for Queue?
  • Already known to students
  • Easy to visualise
  • Allowed and acceptable at 9618 level

We treat the front of the queue as index 0.

3. Creating an Empty Queue 
queue = []

An empty list represents an empty queue.

4. ENQUEUE (Add an item)

ENQUEUE means adding an item to the rear of the queue.

queue.append("A")
queue.append("B")

Queue now becomes: ["A", "B"]

5. DEQUEUE (Remove an item)

DEQUEUE means removing the item at the front.

item = queue.pop(0)

The first item is removed.

Queue becomes: ["B"]

6. PEEK (View front item)

PEEK means looking at the front item without removing it.

front_item = queue[0]
7. Checking if Queue is Empty 
if len(queue) == 0:
    print("Queue is empty")
8. Full Queue? (Exam Note)

With Python lists, we normally do not worry about full queues.

In exams, mention: memory limitation defines full queue.
9. Complete Queue Program (Exam‑Friendly)

This is a full working program using Python LIST, suitable for Cambridge 9618 answers.

# Creating an empty queue
queue = []

# ENQUEUE operation
queue.append("Ali")
queue.append("Sara")
queue.append("John")

# Printing queue after ENQUEUE
print("Queue after adding items:", queue)

# DEQUEUE operation
if len(queue) > 0:
    removed_item = queue.pop(0)
    print("Removed item:", removed_item)
else:
    print("Queue is empty, cannot remove")

# Printing queue after DEQUEUE
print("Queue after deletion:", queue)
Explain line‑by‑line in class. Emphasise append() for ENQUEUE and pop(0) for DEQUEUE.
