16.1 Types of System Software
System software controls hardware and provides a platform for application software. Main categories: Operating Systems, Utility Programs, Library Programs, Translators, Linkers/Loaders and Virtual Machines.
Examples of operating systems: Windows 11, macOS, Linux distributions, Android, iOS.
Teacher Note: Clarify to students that system software runs in the background and is essential — applications cannot run without it.
Mini Exercise: Give one example of a system software and one application software; explain the difference in one sentence.
16.2 Utility Software — Deep Dive
Utility software provides maintenance and optimisation functions which support the operating system. They include file managers, disk utilities, security tools, backup and recovery, compression utilities, and system monitors.
Disk utilities: CHKDSK (checks disk errors), Disk Cleanup (removes temporary files), Defragmenter (HDD optimisation). Security utilities include Antivirus and Firewalls. Backup utilities: full, incremental, differential backups.
Tip: SSDs use wear-leveling; defragmentation is unnecessary and harmful for SSDs.
Teacher Note: Ask students to compare incremental vs differential backups — prepare a 3-mark explanation exercise.
Mini Exercise: Explain in one sentence why incremental backups save time compared to full backups.
16.3 Library Programs — Deep Dive
Library programs are collections of pre-written routines (functions, classes) that programs can call. They improve productivity and reliability by enabling code reuse.
Static libraries (.lib, .a) are linked at compile time. Dynamic/shared libraries (.dll, .so, .dylib) are linked at runtime and can be shared among programs.
Teacher Note: Demonstrate creating a small user-defined library (e.g., math helper) and importing it into two programs.
Mini Exercise: State one advantage and one disadvantage of dynamic libraries.
16.4 Translators — Comprehensive
Translators convert source code into machine code. Main types: Assemblers (assembly → machine), Compilers (high-level → machine for whole program), Interpreters (high-level → execute line-by-line).
C++ typically uses a compiler (produces an executable). Python commonly uses an interpreter (executes line-by-line), often with a bytecode stage.
Teacher Note: Run a live demo — compile a small C program and run it; then run an equivalent Python script to show differences in workflow and error reporting.
Mini Exercise: Provide one advantage and one disadvantage of using an interpreter.
16.5 Linkers, Loaders & Virtual Machines — Comprehensive
A linker combines object files and library code into a single executable. A loader places the executable into RAM and prepares it for execution. A virtual machine emulates a computer system (process VM like JVM, or system VM like VirtualBox).
Static linking embeds library code into the executable; dynamic linking uses shared libraries loaded at runtime. JVM runs Java bytecode, enabling platform independence.
Teacher Note: Illustrate the full build chain: source → compiler/assembler → object files → linker → executable → loader → process (or bytecode → VM).
Mini Exercise: Explain the difference between static and dynamic linking in two sentences.
16.6 Role of an Operating System in Program Execution — Final
The OS manages process creation, CPU scheduling, memory allocation (paging/segmentation/virtual memory), file access, device drivers, interrupts, and security while programs execute.
When a user runs a program: loader brings the executable to memory, OS sets up a PCB, scheduler allocates CPU time, I/O requests are processed by device drivers, and the OS reclaims resources on termination.
Teacher Note: Use a layered diagram (User → Application → OS → Hardware) and perform a Task Manager demo to visualise processes and resource usage.
Mini Exercise: List three items stored in a Process Control Block (PCB).
