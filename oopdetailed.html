Teacher Notes

Introduction to OOP

Object Oriented Programming (OOP) is a paradigm that structures programs using objects which combine data and behaviour.

  • Attributes: represent the state of an object
  • Methods: represent the behaviour of an object

OOP is ideal for modelling real-world systems and is heavily assessed at A Level.

Classes & Objects

A class acts as a blueprint from which objects are created.

class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

Teaching focus: Emphasise the difference between class definition and object instantiation.

Encapsulation

Encapsulation bundles data and methods together while restricting direct access to internal data.

class Account:
    def __init__(self, balance):
        self.__balance = balance

    def get_balance(self):
        return self.__balance

Stress data hiding and exam terminology such as private attributes.

Inheritance & Polymorphism

Inheritance enables code reuse, while polymorphism allows the same method name to behave differently.

class Dog:
    def sound(self):
        print("Bark")

class Cat:
    def sound(self):
        print("Meow")

Abstraction & UML

Abstraction hides implementation details. UML diagrams visually represent class structure and relationships.

Exam tip: students must interpret and convert UML diagrams into code.

Practice

Classes & Objects

  • Create a Book class with title and author
  • Instantiate an object and display its attributes
  • Explain the difference between a class and an object

Encapsulation Tasks

  • Write a class with a private attribute
  • Provide a public getter method
  • Explain why encapsulation improves security

Inheritance Challenge

  • Define a Vehicle parent class
  • Create a Car subclass
  • Override a method in the child class

Polymorphism & Abstraction

  • Explain polymorphism with your own example
  • Describe abstraction using a real-world analogy

Exam Practice

  • Define OOP (2 marks)
  • Explain two advantages of encapsulation (4 marks)
  • Describe how inheritance supports code reuse (4 marks)
