Short Answer Questions (15 Questions)

Short Answer Solutions:

1. Pipelining is instruction-level parallelism where different stages of fetch–decode–execute happen simultaneously for multiple instructions.

2. Advantages: simpler instructions for faster execution; easier pipelining; more registers.

3. The control unit directs CPU operations ensuring each instruction executes correctly.

4. Hard-wired CU uses logic circuits; Microprogrammed CU uses ROM-stored microcode.

5. SIMD executes the same instruction on multiple data streams simultaneously.

6. MIMD executes different instructions on multiple data streams in parallel.

7. Software emulating complete hardware to run multiple OS/applications.

8. Software providing an environment for a specific program (e.g., Java VM).

9. Performance is lower than native execution.

10. SISD is sequential; SIMD executes the same instruction on multiple data streams.

11. IBM Sequoia, Beowulf clusters

12. Small, fixed-length instruction formats.

13. Registers store intermediate results to allow simultaneous instruction stages.

14. CISC has complex multi-cycle instructions and variable-length formats.

15. Allows applications to run as if on real hardware while host OS manages resources.

Scenario-Based Questions (5 Questions)

Scenario Answers:

1. Data hazard: second instruction reads R1 before first writes. Resolved via stalling, forwarding, or register renaming.

2. SIMD executes same instruction across multiple data elements simultaneously, e.g., multiplying all array elements at once.

3. System VM; it emulates hardware to run a different OS on the host for legacy software or multi-OS usage.

4. MIMD executes different instructions on different data streams; SIMD executes same instruction on multiple data streams.

5. Flush partially executed instructions and preserve pipeline states with individual program counters.
